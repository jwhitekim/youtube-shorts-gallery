# YouTube 데스크톱 홈 — 디자인 규칙 (Design System)

> 분석 대상: `https://www.youtube.com/?app=desktop&hl=ko&gl=KR` (라이트 모드, 데스크톱)
> 분석 방법: 메인 홈 화면 스크린샷 1장 시각 분석 + YouTube 공개 CSS 토큰(`--yt-spec-*`) 관행 참조
>
> **표기 규칙**
> - 〔측정〕 — 스크린샷에서 직접 관찰됨
> - 〔추정〕 — 이미지 기반 + 일반적 디자인 관행으로 추론 (px 값 등)
> - 〔불확실〕 — 한 장의 스크린샷만으로는 단정할 수 없음

---

## 1. 디자인 철학 (Design Philosophy)

YouTube 홈 페이지는 다음 세 가지 원칙으로 설계되어 있다.

1. **콘텐츠 우선 (Content-first)** — UI 크롬(헤더, 사이드바, 카드 메타정보)은 무채색·중립 톤으로 억제하고, 컬러는 거의 전적으로 콘텐츠(썸네일, 브랜드 로고)에서만 발생하도록 한다. 사용자의 시각적 주의가 영상 콘텐츠로 향하게 하는 게 목적이다.
2. **밀도 중심 (Density-driven)** — 데스크톱 viewport에 가능한 한 많은 비디오 카드를 노출한다. 한 행에 3개 카드 + Shorts 5개를 동시에 보여주는 다중 그리드 구성을 쓴다.
3. **계층의 시각적 절제 (Restrained hierarchy)** — 강조는 색상이 아니라 **굵기(weight)**, **크기 차이**, **여백**으로만 표현한다. 화면에 등장하는 진한 검정은 거의 유일하게 "전체" 칩의 활성 상태뿐이다.

## 2. 색상 시스템 (Color System)

### 2.1 팔레트 〔측정 + 추정〕

YouTube는 자체 디자인 토큰(`--yt-spec-*`)을 사용한다. 라이트 모드에서 실제 사용되는 값(공개된 관행 기준)은 다음과 같다.

| 역할 | 토큰 (관행) | 값 〔추정〕 | 용도 |
|------|------------|------|------|
| 기본 배경 | `--yt-spec-base-background` | `#FFFFFF` | 페이지 전체 배경, 헤더 배경 |
| 보조 배경 | `--yt-spec-additive-background` | `rgba(0,0,0,0.05)` | 검색창, 카테고리 칩 비활성, 호버 영역 |
| 1차 텍스트 | `--yt-spec-text-primary` | `#0F0F0F` | 비디오 제목, 사이드바 메뉴 |
| 2차 텍스트 | `--yt-spec-text-secondary` | `#606060` | 채널명, 조회수·시간 메타정보, "스폰서 · Contabo" |
| 강조 (Call-to-action) | `--yt-spec-call-to-action` | `#065FD4` 계열 | 링크, 활성 상태 (이 화면에서는 거의 보이지 않음) |
| 활성 칩 배경 | — | `#0F0F0F` (검정) | "전체" 칩 같은 selected chip |
| 활성 칩 텍스트 | — | `#FFFFFF` | 위 칩 위 글자 |
| 브랜드 적색 | YouTube Red | `#FF0000` | YouTube 로고 재생 버튼 삼각형 |

### 2.2 색상 사용 규칙

- **무채색 80% + 콘텐츠 색상 20% 원칙** 〔추론〕 — UI 크롬은 흰색·회색·검정만 사용. 색은 영상 썸네일·로고에서만 등장.
- **그라디언트 사용 안 함** 〔측정〕 — 화면 전체에서 그라디언트가 한 곳도 보이지 않는다. 모든 면은 솔리드 컬러.
- **그림자 사용 거의 안 함** 〔측정〕 — 카드, 버튼, 헤더에 drop shadow가 보이지 않는다. 깊이는 색 대비와 라운드 코너로만 표현된다.

## 3. 타이포그래피 (Typography)

### 3.1 폰트 패밀리 〔추정〕

YouTube는 **Roboto** 패밀리를 영문 기본으로 사용하고, 한국어는 시스템 폰트 스택으로 fallback 한다. CSS 관행:

```css
font-family: "Roboto", "Apple SD Gothic Neo", "Malgun Gothic", "맑은 고딕", sans-serif;
```

### 3.2 타이포 스케일 〔측정 + 추정〕

| 역할 | 크기 〔추정〕 | 굵기 | 줄높이 | 사례 |
|------|------|------|--------|------|
| 비디오 제목 | 14–16 px | **600 (Semibold)** | 1.4 | "연봉 7억 개발자면…" |
| 채널명 | 12–14 px | 400 (Regular) | 1.4 | "비디오머그 - VIDEOMUG" |
| 메타정보 (조회수·시간) | 12–14 px | 400 (Regular) | 1.4 | "조회수 9.5만회 · 12시간 전" |
| 카테고리 칩 | 14 px | 500 (Medium) | 1.0 | "전체", "뉴스", "팟캐스트" |
| 사이드바 메뉴 | 14 px | 400 (Regular) | 1.0 | "홈", "Shorts", "구독" |
| 사이드바 섹션 헤더 | 16 px | 500 (Medium) | 1.0 | "내 페이지", "탐색" |
| 섹션 헤더 (Shorts 등) | 20 px | 700 (Bold) | 1.2 | "Shorts" |
| 헤더 로고 | — (이미지) | — | — | "YouTube" 워드마크 |

### 3.3 위계 표현 규칙

- **위계는 굵기로 표현** 〔측정〕 — 같은 영역 안에서 제목(굵게)과 메타정보(보통)는 거의 동일한 크기지만 굵기로 구분된다.
- **컬러는 보조적 위계 표현** 〔측정〕 — 1차 정보는 `#0F0F0F` (거의 검정), 2차 정보는 `#606060` (중간 회색).
- **줄바꿈 정책**: 비디오 제목은 최대 **2줄**까지 노출하고 그 이상은 ellipsis (`...`)로 자른다. 〔관행 + 측정〕

## 4. 레이아웃 시스템 (Layout System)

### 4.1 전체 구조

```
┌─────────────────────────────────────────────────────────┐
│  Header (고정, sticky, 56px)                              │
├──────────┬──────────────────────────────────────────────┤
│          │                                              │
│ Sidebar  │  Main Content                                │
│  (고정)   │  ├─ Category Chips (sticky 아래쪽 헤더)        │
│          │  ├─ Video Grid (3 cols × N rows)             │
│  240px   │  ├─ Shorts Shelf (가로 스크롤)                 │
│          │  └─ ...                                      │
│          │                                              │
└──────────┴──────────────────────────────────────────────┘
```

### 4.2 고정 영역 〔측정 + 추정〕

| 영역 | 너비/높이 〔추정〕 | 동작 |
|------|------------|------|
| Header | 높이 56 px | `position: sticky; top: 0` |
| Sidebar (펼친 상태) | 너비 240 px | 페이지 스크롤과 별개로 자체 스크롤 |
| Sidebar (접힌 상태) | 너비 72 px | 햄버거 메뉴 클릭 시 토글 |
| Main 좌측 패딩 | 24 px | 사이드바와 콘텐츠 사이 간격 |

### 4.3 비디오 그리드 〔측정 + 추정〕

- **컬럼 수**: viewport 너비에 따라 반응형 변화. 이 스크린샷 너비(약 1400 px)에서 **3열**.
  - 일반적 breakpoint 〔관행〕: ~1300 px 이상 4열, 1000–1300 px 3열, 700–1000 px 2열, 그 이하 1열.
- **카드 간격 (gutter)** 〔추정〕: 가로 16 px, 세로 40 px. 가로보다 세로가 넓다. → 행 단위 시각 분리를 강조.
- **썸네일 비율** 〔측정〕: 16:9 고정.
- **썸네일 라운딩** 〔측정〕: corner radius **12 px** 정도. 비교적 큰 라운딩이 YouTube 2022년 리디자인의 시그니처.

### 4.4 Shorts 셀프 〔측정 + 추정〕

- **카드 비율**: 9:16 세로형 (썸네일 자체).
- **한 줄 노출 개수**: 이 화면에서 5개. 우측에 잘리지 않고 깔끔히 떨어지는 걸 보면 grid 기반이지만 가로 스크롤도 지원함 〔추론〕.
- **카드 라운딩** 〔측정〕: 12 px (일반 비디오 카드와 동일).

## 5. 컴포넌트 (Components)

### 5.1 헤더 (Header)

3분할 구조. 좌측(브랜드+내비) — 중앙(검색) — 우측(액션).

| 영역 | 구성 |
|------|------|
| **좌측** | 햄버거 아이콘 (24px) → YouTube 로고 (높이 ~20px, "KR" superscript) |
| **중앙** | 검색창 (`pill`, 좌측 둥근 input + 우측 둥근 search 버튼) + 음성 검색 마이크 버튼 (원형) |
| **우측** | "+ 만들기" 버튼 (pill, light gray bg) + 알림 종 아이콘 + 사용자 아바타 (원형 32px) |

**검색창 디자인 규칙** 〔측정 + 추정〕:
- 형태: 좌측 직사각 input + 우측 정사각형 search 버튼이 합쳐진 pill 모양
- input 좌측 borderRadius `40px`, 우측 `0`
- search 버튼 좌측 borderRadius `0`, 우측 `40px`
- input border `1px solid #cccccc`, focus 시 `#1c62b9` 근처로 변경 〔관행〕
- search 버튼 배경 `#f8f8f8`, 호버 시 `#f0f0f0` 〔관행〕
- input 좌측 안쪽에 placeholder "검색"
- 가상 키보드 아이콘이 input 우측에 inline으로 노출 〔측정〕

**"+ 만들기" 버튼**:
- pill 형태, `border-radius: 18px`
- 배경 매우 옅은 회색 (`#f2f2f2` 정도) 〔추정〕
- 좌측에 `+` 아이콘 + 우측 텍스트 "만들기"
- padding 가로 12 px, 세로 8 px 〔추정〕

### 5.2 사이드바 (Sidebar)

수직 메뉴. 시각적으로 4개 섹션으로 구분된다.

| 섹션 | 항목 | 시각 처리 |
|------|------|-----------|
| **상단 (구분선 없음)** | 홈, Shorts, 구독 | 첫 항목 "홈"이 활성 상태 → 배경 옅은 회색 채움 |
| **내 페이지 ›** | 내 채널, 기록, 재생목록, 나중에 볼 동영상, 좋아요 표시한 동영상, 내 동영상, 오프라인 저장 동영상, 더보기 | 섹션 헤더는 작은 텍스트 + 우측 `›` 화살표 (확장 가능 시사) |
| **탐색** | 쇼핑, 음악, 영화, 더보기 | 위와 동일 패턴 |
| **YouTube 더보기** (잘림) | — | 화면 하단으로 잘림 |

**메뉴 아이템 디자인 규칙** 〔측정 + 추정〕:
- 좌측 아이콘 (24×24) + 우측 텍스트, 간격 24 px
- 행 높이 약 40 px
- 좌우 패딩 12 px
- 활성 상태: 배경 `#f2f2f2`, 행 전체 둥근 사각형 `border-radius: 10px`
- 호버 상태: 위와 유사한 배경, 배경색만 약간 옅음 〔관행〕
- 아이콘은 모두 **outlined** 스타일 (filled 아님). 단, 활성 상태인 "홈"만 filled로 전환된다 〔측정〕.

### 5.3 카테고리 칩 (Category Chips)

메인 콘텐츠 최상단의 가로 스크롤 가능한 필터.

**디자인 규칙** 〔측정 + 추정〕:
- 형태: pill, `border-radius: 8px` (완전 라운드 아님)
- 비활성: 배경 `#f2f2f2`, 텍스트 `#0f0f0f`
- 활성("전체"): 배경 `#0f0f0f`, 텍스트 `#ffffff`
- padding 가로 12 px, 세로 8 px 〔추정〕
- 칩 간격 12 px
- 폰트 14 px / 500 weight

이 컴포넌트가 화면에서 거의 유일한 "검정 채움" 영역이다. → 활성 필터를 강조하기 위한 의도적 선택.

### 5.4 비디오 카드 (Video Card)

표준 카드 구조:

```
┌─────────────────────────┐
│                         │
│      썸네일 (16:9)        │  ← border-radius 12px
│                  [14:04]│  ← 영상 길이 우하단
└─────────────────────────┘
 ┌──┐  제목 (2줄까지)        │  ← Semibold, 14–16px
 │채│  채널명 ✓             │  ← Regular, secondary color
 │널│  조회수 9.5만회 · 12시간 전│
 └──┘                      ⋮  ← 우측 더보기(kebab)
```

**디자인 규칙** 〔측정 + 추정〕:
- 썸네일 라운딩 12 px
- 썸네일 우하단에 영상 길이 라벨 (배경 반투명 검정 `rgba(0,0,0,0.8)`, 텍스트 흰색, 12px, 작은 패딩)
- 메타 영역에 채널 아바타 (원형 36×36) + 텍스트 컬럼 + kebab 메뉴
- 제목과 메타 사이 여백 12 px 〔추정〕
- 카드 자체에는 border, shadow, 배경색이 없다. 〔측정〕 → 배경에 그대로 떠 있는 형태.
- 인증 채널은 채널명 옆에 `✓` 배지 (작은 회색 체크). 〔측정〕

**스폰서 카드 차이점** 〔측정〕:
- "Contabo.Rock-Solid Performance" 카드는 영상 길이 라벨 자리에 우하단 화살표 아이콘 (외부 링크 시사)
- 메타 영역에 채널 아바타가 없고 좌측에 "스폰서 · Contabo" 텍스트
- 채널명·조회수 자리에 광고 디스크립션 2줄
- → 광고 카드는 동일한 카드 폼팩터를 쓰되 **메타 영역의 정보 종류**만 바꿔서 일반 영상과 시각적으로 통일

### 5.5 Shorts 카드 (Shorts Card)

**디자인 규칙** 〔측정 + 추정〕:
- 비율 9:16 세로형
- 라운딩 12 px (일반 카드와 동일)
- 메타 정보가 카드 외부가 아니라 **썸네일 안 하단 오버레이**에 표시되는 경우가 많다 〔측정〕 — 이 스크린샷에선 썸네일만 보이고 텍스트는 잘려 있어 단정 어려움 〔불확실〕
- 셀프 헤더에 빨간 Shorts 로고 + "Shorts" 텍스트 + 우측 kebab 메뉴

## 6. 인터랙션 & 상태 (Interaction States) 〔관행 기반, 일부 불확실〕

이 스크린샷은 정적 화면이라 호버·포커스 상태는 **직접 관찰 불가**. YouTube의 일반적 패턴은 다음과 같다.

| 컴포넌트 | 기본 | 호버 | 활성/선택 |
|----------|------|------|-----------|
| 사이드바 메뉴 | 투명 배경 | `#f2f2f2` 배경 | `#f2f2f2` + 아이콘 filled |
| 카테고리 칩 | `#f2f2f2` | 약간 어두운 회색 | `#0f0f0f` + 흰 텍스트 |
| 비디오 카드 썸네일 | 정적 이미지 | 자동재생 미리보기 (수 초 후) | — |
| 카드 kebab 메뉴 | hidden / faded | 표시됨 | 클릭 시 드롭다운 |
| 헤더 아이콘 버튼 | 투명 배경 | 원형 배경 `#f2f2f2` | — |

## 7. 아이콘 시스템 (Iconography) 〔측정 + 추정〕

- **스타일**: outlined (선) 기본, filled은 활성 상태만.
- **크기**: 사이드바·헤더 모두 24×24 px가 표준. 채널 인증 배지만 12 px.
- **출처**: Google Material Symbols 계열 〔관행〕.
- **stroke**: 약 2 px, line-cap rounded 〔추정〕.

## 9. 접근성 / 다국어 〔측정〕

- 한국어 UI지만 폰트는 영문(Roboto)과 한글이 시스템 폰트로 자연스럽게 섞임. → CJK 폰트가 별도로 적재되지 않고 OS 의존.
- "KR" 슈퍼스크립트가 로고 우상단에 붙어 현재 지역 표시.
- 사이드바 모든 메뉴 항목이 텍스트 라벨을 가짐 (아이콘만 쓰지 않음).

## 10. 핵심 디자인 결정 요약 (Key Design Decisions)

이 페이지를 클론할 때 반드시 지켜야 할 7가지:

1. **순백 배경 + 검정·회색 2단계 텍스트** — 색은 콘텐츠에서만 발생.
2. **모든 카드는 그림자 없이 라운딩(12px)만으로 구분**.
3. **위계는 굵기로**, 색이나 크기 차이는 보조 수단.
4. **세로 간격이 가로 간격보다 크다** (40px > 16px) — 행 분리 강조.
5. **활성 상태는 검정 채움 + 흰 텍스트** — 칩에서만 사용.
6. **아이콘은 outlined가 기본, 활성만 filled** — 사이드바 "홈"이 유일.
7. **검색창은 input + 버튼이 합쳐진 비대칭 pill** — 좌우 라운딩이 다른 게 시그니처.

## 12. 한계 (Limitations of this Analysis)

- 한 장의 정적 스크린샷 기반이라 **호버·포커스·로딩·에러 상태**는 추론에만 의존.
- **다크 모드**의 색상 토큰은 별도 분석 필요.
- 정확한 px 값은 브라우저 줌 비율을 모르므로 〔추정〕. 실제 구현 시 DevTools에서 `--yt-spec-*` CSS 변수를 직접 읽는 것이 가장 정확하다.
- **모바일 레이아웃**, **비디오 재생 페이지**, **검색 결과 페이지**는 이 분석에 포함되지 않음.
